# embedder-pipedocs-court

Stage 2 output of [`module-embedder`](https://github.com/ai-pipestream/module-embedder) on the CourtListener 1000-doc corpus. Each PipeDoc is embedded with **two** models (`minilm` and `paraphrase-minilm`) and packaged as classpath resources for downstream pipeline integration tests that need representative embedder output without running the embedder against a real DJL Serving endpoint.

Standalone repo (not a subproject of `sample-documents`) because the total artifact size (~835 MB) would crowd Sonatype Central Portal's ~1 GB per-upload limit when bundled with other fixture data modules.

## Artifact coordinates

- **GroupId**: `ai.pipestream`
- **ArtifactId**: `embedder-pipedocs-court`
- **Version**: axion-release from git tags (`v`-prefixed)

## Layout

Each PipeDoc is a single non-delimited `ai.pipestream.data.v1.PipeDoc` serialised with `writeTo(GZIPOutputStream)` — one file per doc. Matches the per-doc-per-file pattern of `sample-documents/double-chunked-pipedocs`. Every file is well under GitHub's 100 MB per-file limit (largest single file: ~16 MB, most are 100-200 KB).

```
src/main/resources/fixtures/court/
  ├── stage2_sentence_1000/        # 1000 docs, sentence-chunked, 2 SPRs/doc (minilm + paraphrase-minilm)
  │     └── doc_0001.pb.gz ... doc_1000.pb.gz        (1000 files ~186 MB)
  ├── stage2_token_1000/           # 1000 docs, token-chunked, 2 SPRs/doc
  │     └── doc_0001.pb.gz ... doc_1000.pb.gz        (1000 files  ~99 MB)
  ├── stage2_paragraph_1000/       # 1000 docs, paragraph-chunked, 2 SPRs/doc
  │     └── doc_0001.pb.gz ... doc_1000.pb.gz        (1000 files ~169 MB)
  ├── stage2_sentence_full_100/    # 100 docs, sentence-chunked + sentences_internal, 4 SPRs/doc
  │     └── doc_0001.pb.gz ... doc_0100.pb.gz        (100 files ~136 MB)
  ├── stage2_token_full_100/       # 100 docs, token-chunked + sentences_internal, 4 SPRs/doc
  │     └── doc_0001.pb.gz ... doc_0100.pb.gz        (100 files ~121 MB)
  └── stage2_paragraph_full_100/   # 100 docs, paragraph-chunked + sentences_internal, 4 SPRs/doc
        └── doc_0001.pb.gz ... doc_0100.pb.gz        (100 files ~134 MB)
```

Total: 3300 files, ~842 MB.

## Two variant types

| Suffix       | Docs | SPRs per doc | Chunker configs                              | Includes `sentences_internal` |
|--------------|------|--------------|----------------------------------------------|-------------------------------|
| `_1000`      | 1000 | 2            | user variant only (sentence / token / paragraph) | no                            |
| `_full_100`  | 100  | 4            | user variant + `sentences_internal`          | yes                           |

The `_full_100` variants include the chunker's `sentences_internal` SPR (per-sentence chunks, ~10× the chunk count of token / paragraph) for consumers that need embedded sentence-granular results — coreference, fine-grained NLP, or cross-SPR similarity testing. Kept at 100 docs so the fixture stays tractable.

## Reading from the classpath

```java
import ai.pipestream.data.v1.PipeDoc;
import java.io.InputStream;
import java.util.zip.GZIPInputStream;

try (InputStream in = getClass().getResourceAsStream("/fixtures/court/stage2_token_1000/doc_0042.pb.gz");
     GZIPInputStream gz = new GZIPInputStream(in)) {
    PipeDoc doc = PipeDoc.parseFrom(gz);
    // doc.getSearchMetadata().getSemanticResultsList() has:
    //   - 2 SPRs (one per model) for the _1000 variants
    //   - 4 SPRs (2 chunkers × 2 models) for the _full_100 variants
    // every SPR's chunks carry embedding_info.vector (384 floats) populated
}
```

To iterate every doc in a variant, use the stable `doc_NNNN.pb.gz` naming:

```java
for (int i = 1; i <= 1000; i++) {
    String path = String.format("/fixtures/court/stage2_token_1000/doc_%04d.pb.gz", i);
    try (InputStream in = getClass().getResourceAsStream(path);
         GZIPInputStream gz = new GZIPInputStream(in)) {
        PipeDoc doc = PipeDoc.parseFrom(gz);
        // ...
    }
}
```

## Source

Generated by `module-embedder`'s `EmbedderFixtureDumpIT` against the sibling [`chunker-pipedocs-court`](https://github.com/ai-pipestream/sample-documents) artifact as stage-1 input, embedding against real DJL Serving. Regenerate both variants via:

```bash
cd module-embedder

# Filtered 1000-doc (no sentences_internal)
./gradlew :module-embedder:quarkusIntTest -PrunFixtureDump \
    -PfixtureOutDir=/tmp/embedder-fixtures \
    -PfixtureOpinions=1000

# Full 100-doc (with sentences_internal, 4 SPRs/doc)
./gradlew :module-embedder:quarkusIntTest -PrunFixtureDump \
    -PfixtureOutDir=/tmp/embedder-fixtures \
    -PfixtureOpinions=100 -PfixtureKeepInternalSentences=true
```

Each run produces 3 big length-delimited `.pb.gz` streams (`stage2_{variant}_{N}.pb.gz`). A post-process splitter (Java program, see git history) breaks each stream into per-doc files at doc boundaries before commit.

Total embedder runtime: ~2.5 min against DJL on krick (1m 38s for filtered 1000-doc + 56s for full 100-doc).
